## 2026-06-09

- **Added Garden Seasons system with calendar-based meteorological seasons**: The garden now feels the turn of real-world seasons derived from `new Date().getMonth()` — spring (Mar-May), summer (Jun-Aug), autumn (Sep-Nov), winter (Dec-Feb). Each season applies a CSS-only overlay class on `<body>` that layers on top of the existing time-of-day themes, adding season-specific atmospheric tints and animated particle effects: spring drifts pink petal motes through the visitors layer, summer adds warm golden shimmer particles, autumn brings drifting amber leaf particles, and winter scatters slow-falling snow crystals. Each season can trigger one rare transient event per garden visit — spring's morning dew adds dew-drop sparkles to all bloomed flowers or spawns extra butterflies; summer's golden shimmer intensifies flower glow on a random blooming tile; autumn's harvest breeze scatters leaves from wilted tiles; winter's first frost briefly frosts all tile borders. Seasonal state persists via localStorage (last recorded month/season), and a journal entry notes the seasonal shift the first time it's detected on load. A season indicator (🌷/🌻/🍂/❄️) replaces the time-of-day emoji in the stats panel. Files changed: `docs/js/theme.js` (added `getCurrentCalendarSeason()` export), `docs/js/garden-seasons.js` (new module), `docs/styles.css` (seasonal overlays + rare event keyframes), `docs/js/stats.js` (garden season display), `docs/script.js` (import + init), `docs/index.html` (season stat card id). All effects are pure CSS animations, respect `prefers-reduced-motion`, use relative units, and follow the dark nature palette.

## 2026-06-09

- **Added off-screen garden aging system**: When returning to the garden after time away, the garden now simulates weather cycles, volunteer wildflower spawns, growth progression, and a welcome-back experience — directly addressing the VISION.md promise that 'every return visit should feel like coming home to something that has continued growing in your absence.' The `garden-aging.js` module records a `lastVisitTimestamp` to localStorage on page unload, computes elapsed hours on load, and for every ~2 hours elapsed simulates one weather state change cycling through sunny→cloudy→rainy. If 6+ hours elapsed with 3+ blooming flowers, a volunteer wildflower sprouts on a random empty tile. If 24+ hours elapsed, a random planted tile advances one growth cycle. If 48+ hours elapsed, a poetic welcome-back journal entry summarizes what changed. A CSS-animated welcome-back banner overlays the viewport with the elapsed time and a poem about the garden's journey. All simulated changes trigger journal entries so the user sees a timeline of what 'happened.' Simulation caps at 7 days of elapsed time to prevent absurd states. All changes use only existing systems (weather cycling, self-seeding, growth cycles, journal), CSS-only animations for the banner, respect `prefers-reduced-motion`, use relative units, and keep the dark nature palette.
